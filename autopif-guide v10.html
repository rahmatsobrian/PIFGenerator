Argumen & Opsi

--product
Pilih device langsung tanpa menu interaktif.
Contoh: --product shiba_beta atau --product shiba (suffix _beta otomatis ditambah).
-o / --output
Ubah path/nama file output.
Default: pif.prop di direktori aktif. Contoh: -o /sdcard/Download/mypif.prop
--existing
Baca spoof settings dari pif.prop lama.
Berguna agar setting kustom tidak perlu diisi ulang. Contoh: --existing pif.prop.bak
--auto
Skip pertanyaan spoof settings, langsung pakai nilai default recommended tanpa input manual. Cocok untuk scripting.
--list / -l
Tampilkan daftar device yang tersedia dalam format JSON lalu keluar. Cocok untuk parsing otomatis.
-h / --help
Tampilkan bantuan singkat semua opsi yang tersedia langsung di terminal.
CONTOH KOMBINASI ARGUMEN # Pilih device + output custom + skip pertanyaan bash pifgenerator.sh --product shiba_beta -o /sdcard/pif.prop --auto # Pakai spoof settings dari file lama bash pifgenerator.sh --existing pif.prop.bak --auto # Lihat daftar device tersedia (format JSON) bash pifgenerator.sh --list # Kombinasi lengkap: device + existing + auto bash pifgenerator.sh --product rango --existing pif.prop -o pif_baru.prop --auto

Alur Proses Script

1

Crawl Daftar Device

Akses developer.android.com untuk mendapat daftar Pixel dengan build Beta/Canary terbaru.

2

Pilih Device

Menu bernomor tampil. Ketik nomor pilihan atau gunakan --product untuk skip langsung. Ketik 0 untuk keluar.

3

Ambil Build Canary via Flash Station

Akses Flash Station API Google untuk mendapat Release ID dan Incremental dari build Canary terbaru device tersebut.

4

Ambil Security Patch

Crawl Pixel Bulletin di source.android.com untuk tanggal patch. Jika gagal, estimasi otomatis dari Release ID.

5

Konfigurasi Spoof Settings

Tekan Enter untuk pakai default, e untuk edit manual. Gunakan --auto untuk skip sepenuhnya.

6

Pilih Format Output

[1] pif.prop  ·  [2] pif.json  ·  [3] keduanya

7

Simpan & Selesai

Simpan ke /sdcard/Download/, folder aktif, atau keduanya. File tmp otomatis dibersihkan.

🛡️

Spoof Settings

Setiap setting adalah toggle 1 (aktif) atau 0 (nonaktif). Nilai default sudah optimal untuk lulus Play Integrity.

spoofBuild
Spoof nilai Build.* ke nilai Pixel Canary yang digenerate.
Default: 1 (aktif)
spoofProps
Spoof system props seperti ro.product.* dan ro.build.*
Default: 1 (aktif)
spoofProvider
Spoof ContentProvider untuk verifikasi integritas tambahan.
Default: 1 (aktif)
spoofSignature
Spoof signature certificate build ke signature Pixel asli.
Default: 0 (nonaktif)
spoofVendingFinger
Spoof fingerprint khusus untuk Google Play Store.
Default: 1 (aktif)
spoofVendingSdk
Spoof SDK level untuk Play Store, membantu kecocokan app.
Default: 0 (nonaktif)
Sudah punya pif.prop lama dengan setting kustom? Gunakan --existing pif.prop.lama agar setting tersebut dibaca otomatis.

Pertanyaan Umum

Pastikan koneksi internet aktif dan tidak ada VPN/firewall yang memblokir developer.android.com atau flash.android.com. Coba jalankan pkg install curl untuk memastikan curl tersedia dan up-to-date.
Default tersimpan di /sdcard/Download/pif.prop. Jika sdcard tidak bisa diakses, otomatis disimpan di folder aktif Termux. Gunakan -o /path/kamu/pif.prop untuk menentukan lokasi sendiri.
Tidak perlu root sama sekali! Script ini hanya menggenerate file pif.prop/pif.json. Yang memerlukan root adalah proses instalasi modul PlayIntegrityFix ke sistem Android.
pif.prop adalah format key=value yang digunakan PlayIntegrityFix (KOWX712/chiteroman). pif.json adalah format JSON yang digunakan PlayIntegrityFork (osm0sis) dan beberapa modul alternatif. Pilih sesuai modul yang kamu pakai.
Disarankan generate ulang setiap kali Google merilis build Canary baru (biasanya 1–4 minggu sekali). Tanda harus update: Play Integrity mulai gagal atau muncul pesan "device not certified".
Build Canary adalah build pre-release terbaru dari Pixel. Fingerprintnya biasanya belum masuk daftar revocation Google, sehingga lebih efektif untuk spoofing Play Integrity dibanding build stabil yang sudah lebih "dikenal" sistem.
Google mungkin mengubah struktur halaman developer.android.com. Coba jalankan ulang beberapa saat kemudian, atau tunggu update script. Bisa juga karena koneksi terputus saat proses crawling berlangsung.
Gunakan kombinasi --product, --auto, dan -o untuk fully non-interactive. Contoh: bash pifgenerator.sh --product shiba --auto -o /sdcard/pif.prop. Hasilkan JSON output dengan --list untuk parsing daftar device secara programatik.
